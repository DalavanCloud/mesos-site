42 // The scheme an external containerizer programs have to adhere to is;
43 //
44 // COMMAND < INPUT-PROTO > RESULT-PROTO
45 //
46 // launch < containerizer::Launch
47 // update < containerizer::Update
48 // usage < containerizer::Usage > mesos::ResourceStatistics
49 // wait < containerizer::Wait > containerizer::Termination
50 // destroy < containerizer::Destroy
51 // containers > containerizer::Containers
52 // recover
53 //
54 // 'wait' on the external containerizer side is expected to block
55 // until the task command/executor has terminated.
56 //
57 // Additionally, we have the following environment variable setup
58 // for external containerizer programs:
59 // MESOS_LIBEXEC_DIRECTORY = path to mesos-executor, mesos-usage, ...
60 // MESOS_WORK_DIRECTORY = slave work directory. This should be used
61 // for distiguishing slave instances.
62 // MESOS_DEFAULT_CONTAINER_IMAGE = default image as provided via
63 // slave flags (default_container_image). This variable is provided
64 // only in calls to 'launch'.
65 
66 // Check src/examples/python/test_containerizer.py for a rough
67 // implementation template of this protocol.
68
