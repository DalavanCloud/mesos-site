Detailed Description

Interface to Mesos for a scheduler.

Abstracts master detection (connection and disconnection) and authentication if some credentials are provided.

Expects three callbacks, 'connected', 'disconnected', and 'received' which will get invoked serially when it's determined that we've connected, disconnected, or received events from the master. Note that we drop events while disconnected but it's possible to receive a batch of events across a disconnected/connected transition before getting the disconnected and then connected callback.
