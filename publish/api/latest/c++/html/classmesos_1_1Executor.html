Detailed Description

Callback interface to be implemented by frameworks' executors.

Note that only one callback will be invoked at a time, so it is not recommended that you block within a callback because it may cause a deadlock.

Each callback includes a pointer to the executor driver that was used to run this executor. The pointer will not change for the duration of an executor (i.e., from the point you do ExecutorDriver::start() to the point that ExecutorDriver::join() returns). This is intended for convenience so that an executor doesn't need to store a pointer to the driver itself.

Invoked when a task running within this executor has been killed (via SchedulerDriver::killTask).

Note that no status update will be sent on behalf of the executor, the executor is responsible for creating a new TaskStatus (i.e., with TASK_KILLED) and invoking ExecutorDriver::sendStatusUpdate.

Implemented in BalloonExecutor, LongLivedExecutor, TestExecutor, mesos::python::ProxyExecutor, and JNIExecutor.

virtual void mesos::Executor::shutdown ( ExecutorDriver *  driver)
pure virtual

Invoked when the executor should terminate all of its currently running tasks.

Note that after a Mesos has determined that an executor has terminated any tasks that the executor did not send terminal status updates for (e.g., TASK_KILLED, TASK_FINISHED, TASK_FAILED, etc) a TASK_LOST status update will be created.

Implemented in BalloonExecutor, LongLivedExecutor, TestExecutor, mesos::python::ProxyExecutor, and JNIExecutor.
