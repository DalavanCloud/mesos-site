51 // We use the following notations throughout the cgroups code. The notations
52 // here are derived from the kernel documentation. More details can be found in
53 // <kernel-source>/Documentation/cgroups/cgroups.txt.
54 //
55 // Hierarchy - A hierarchy contains a set of cgroups arranged in a tree such
56 // that every task in the system is in exactly one of the cgroups
57 // in the hierarchy. One or more subsystems can be attached to a
58 // hierarchy.
59 // Subsystem - A subsystem (e.g. cpu, memory, cpuset, etc) in the kernel. Each
60 // subsystem can be attached to only one hierarchy.
61 // Cgroup - A cgroup is just a set of tasks with a set of controls for one
62 // or more subsystems.
63 // Control - A control file in a cgroup (e.g. tasks, cpu.shares).
