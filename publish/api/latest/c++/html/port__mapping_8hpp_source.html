114 // Provides network isolation using port mapping. Each container is
115 // assigned a fixed set of ports (including ephemeral ports). The
116 // isolator will set up filters on the host such that network traffic
117 // to the host will be properly redirected to the corresponding
118 // container depending on the destination ports. The network traffic
119 // from containers will also be properly relayed to the host. This
120 // isolator is useful when the operator wants to reuse the host IP for
121 // all containers running on the host (e.g., there are insufficient
122 // IPs).
