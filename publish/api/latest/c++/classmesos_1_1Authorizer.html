Detailed Description

An interface for authorization of actions with ACLs.

Refer to "authorizer.proto" and "docs/authorization.md" for the details regarding the authorization mechanism.

Each authorize() function returns Future<bool>, which has the same meaning for all functions. If the action is allowed, the future is set to true, otherwise to false. A third possible outcome is that the future fails, which indicates that the request could not be completed at the moment. This may be a temporary condition.

NOTE: Any request allows bundling multiple values for each entity, which are often principals. Though the default authorizer implementation (LocalAuthorizer) supports this feature, Mesos code currently does not authorize multiple principals in a single call.
